# ?? Visual Design Guide

## Color Palette

### Primary Colors
- **Deep Blue**: `#4A90E2` - Primary buttons and accents
- **Success Green**: `#5CB85C` / `#27AE60` - Win states, positive actions
- **Danger Red**: `#D9534F` / `#E74C3C` - Hangman, wrong guesses, exit
- **Dark Header**: `#2C3E50` - Headers and titles
- **Secondary Dark**: `#34495E` - Footer and secondary elements

### Background Gradients
- **Start Screen**: Blue gradient (`#4FACFE` ? `#00F2FE`)
- **Game Screen**: Purple gradient (`#667EEA` ? `#764BA2`)

### UI Elements
- **Card Background**: White with subtle drop shadow
- **Disabled State**: `#CCCCCC` with `#666666` text
- **Borders**: `#E0E0E0` for subtle separations

## Typography

### Fonts
- **Headings**: System default, Bold
- **Word Display**: Consolas (monospace) for consistent letter spacing
- **UI Text**: System default, Regular/SemiBold

### Sizes
- **Main Title**: 48px Bold
- **Section Headers**: 24px SemiBold  
- **Word Display**: 40px Bold with letter-spacing
- **Buttons**: 16-22px SemiBold
- **Body Text**: 16-18px Regular

## Component Styles

### Buttons

#### Modern Button (Primary)
- Background: `#4A90E2`
- Hover: `#357ABD`
- Pressed: `#2868A8`
- Border Radius: 8px
- No border
- Drop shadow: subtle
- Cursor: Hand (pointer)

#### Success Button (Green)
- Background: `#5CB85C`
- Hover: `#4CAF50`
- Used for: Start Game, Play Again

#### Danger Button (Red)
- Background: `#D9534F`
- Hover: `#C9302C`
- Used for: Exit Game

#### Letter Buttons
- Size: 50x50px
- Background: `#ECEFF1` (light grey)
- Hover: `#CFD8DC`
- Disabled: `#F5F5F5` with grey text
- Font: 20px Bold
- Margin: 3px
- Border Radius: 8px

### Cards
- Background: White
- Border Radius: 12px
- Padding: 20px
- Drop Shadow: `rgba(0,0,0,0.2)` with 10px blur, 3px depth

### Input Controls (ComboBox)
- Height: 45px
- Border: 2px solid `#4A90E2`
- Font Size: 18px
- Padding: 10px 8px

## Hangman Drawing Progression

The hangman is drawn on a Canvas (200x250px) with the following stages:

**Always Visible:**
- Gallows structure (brown `#8B4513`)

**Progressive (Red `#E74C3C`):**
1. Head (circle)
2. Body (vertical line)
3. Left arm
4. Right arm
5. Left leg
6. Right leg
7. Left eye (dot)
8. Right eye (dot)
9. Sad mouth (curved down)
10. Tear (dramatic finish!)

## Animations & Interactions

### Button Hover
- Smooth color transition (0.2s)
- Scale: 1.02 (subtle lift effect)
- Cursor changes to pointer

### Card Entrance
- Fade in from 0 to 1 opacity
- Scale from 0.95 to 1.0
- Duration: 0.3s

### Letter Button Click
- Pressed state color change
- Button becomes disabled after use
- Greyed out appearance

### Game Over Overlay
- Background darkens with fade
- Card slides in from top
- Score animates up (count effect)

## Responsive Behavior

### Minimum Size (800x600)
- All elements remain visible
- Keyboard may scroll slightly
- Hangman scales proportionally

### Optimal Size (1200x800)
- Perfect spacing
- No scrolling needed
- Clear visual hierarchy

### Large Screens (1920x1080+)
- Cards have max-width constraints
- Centered layout
- Extra padding/margins

## Accessibility Features

- High contrast text on backgrounds
- Large touch targets (50x50px min for buttons)
- Keyboard navigation support
- Clear focus indicators
- Readable font sizes (16px minimum)
- Color-blind friendly (not relying on color alone)

## Sound Integration

### Sound Events
- ? Correct Guess: Gentle beep (`SystemSounds.Beep`)
- ? Incorrect Guess: Alert (`SystemSounds.Hand`)
- ?? Win: Celebration (`SystemSounds.Exclamation`)
- ?? Lose: Sad sound (`SystemSounds.Asterisk`)

### Sound Control
- Mute button visible on all screens
- Icon changes: ?? (unmuted) ?? ?? (muted)
- State persists during session

## Mobile/Touch Considerations

- All buttons are touch-friendly (minimum 44x44px)
- Virtual keyboard provides full alphabet
- Physical keyboard still works
- No hover-dependent features
